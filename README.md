# 飞书实验室管理 Skill

> 课题组成员、方向、周报的轻量管理方案。
> 数据分两类：结构化的（成员、方向）走 Bitable；长文 markdown（周报）走 Wiki。

📘 **完整规范**：[SKILL.md](./SKILL.md)

## 这是什么

一个 OpenClaw skill，封装了 **飞书 Wiki + Bitable** 双轨管理的初始化流程与文档关系。

适用场景：高校课题组、轻量团队，需要：
- 成员/研究方向的结构化数据（带去重/筛选/统计）
- 周报收集与归档（按起止日期）
- 触发式自动化（群里 @ agent 一句话归档）

## 快速上手

```bash
# 1. 创建你自己的飞书知识库（1 个 Wiki + 1 个 Bitable）
# 2. 记录 4 个 node_token：周报存档 / 方向 / 周报收集 / Bitable 入口
# 3. 读 [SKILL.md](./SKILL.md) §3 初始化流程，逐步配置
# 4. 在 OpenClaw 中加载 skill，群里 @ 触发归档
```

## 核心设计

- **2 表 Bitable**：成员管理（5 字段）+ 方向管理（3 字段）
- **3 节点 Wiki**：周报存档 / 方向 / 周报收集
- **触发式归档**：`请收 [人名] 的周报（起止日期 YYYY-MM-DD 到 YYYY-MM-DD）`
- **自动生成空白周报**：agent 为每个被收周报的成员创建预填模板
- **年级别归档**：`周报存档 / [年份] / [成员名] / [起止日期]`

## 路线

- **v1.x**（已废弃）：3 表 Bitable + Wiki 嵌入块
- **v2.0**：2 表 Bitable + Wiki 收集 + 状态机驱动归档
- **v2.1**：起止日期由触发指令显式指定；自动生成空白周报
