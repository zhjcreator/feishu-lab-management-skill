# 飞书实验室管理 Skill

> 课题组成员、方向、周报的轻量管理方案。
> 数据分两类：结构化的（成员、方向）走 Bitable；长文 markdown（周报）走 Wiki。

---

## 1. 架构总览

```
飞书 Wiki 知识库
│
├── 📁 周报存档
│   └── 📁 [年份]/                    ← **年级别节点**（如 "2026 年"、"2025 年"）
│       └── 📁 [成员名]/
│           └── 📄 [起止日期]   ← 一篇周报（一个 docx）
│
├── 📁 方向
│   └── 📄 [方向名]
│
├── 📁 周报收集（临时区）
│   └── 📄 周报模板
│   └── 📄 [成员名]   ← 当前周期要填写的周报（agent 自动创建）
│
└── 📊 实验室管理（bitable）
    ├── 📋 成员管理
    └── 📋 方向管理
```

## 2. 周报收交流程（起止日期驱动）

### 2.1 触发

群里 @ agent：

```
请收 [人名1]、[人名2] 的周报（起止日期 YYYY-MM-DD 到 YYYY-MM-DD）
```

**示例**：
```
请收张三、李四的周报（起止日期 2026-06-22 到 2026-06-28）
```

**触发者白名单**（默认）：导师、owner。其他人员需 owner 临时授权。

### 2.2 流程

```
[1] 解析触发指令
    └─ 提取人名列表 [张三, 李四]
    └─ 提取起止日期 → 标准化为 YYYY-MM-DD_YYYY-MM-DD
    └─ 年份 = 起始日期的年

[2] 归档当前「周报收集」下所有非模板子页
    a) 列出「周报收集」节点下所有子页 P
    b) 对每个 P：
       - 提交人 = 节点元数据 / 文档内"姓名"段
       - 起止日期 = 文档内"起止日期"段
       - 年份 = 起始日期的年
       - 移动 P → 周报存档 / [年份] / [成员名] / [起止日期]

[3] 自动为每个被收周报的成员创建空白周报
    a) 对每个人名 M：
       - 在「周报收集」下创建子页 M
       - 内容 = 模板（预填起止日期，状态=未完成）
       - 父节点 = 周报收集

[4] 通知提交人
    群里 @ 张三、李四：
    "请在 周报收集 节点下填写你的周报（已预填起止日期 2026-06-22_2026-06-28）"
```

### 2.3 关键设计点

- **起止日期显式指定**：从触发指令里提取，不由状态机自动推算
- **目录按年分组**：周报存档下按年份分子目录
- **自动生成空白周报**：成员复制模板的工作由 agent 自动化
- **状态机被简化**：不再需要"上次要求时间"字段

## 3. 周报模板

```markdown
> **起止日期**：YYYY 年 M 月 D 日 - YYYY 年 M 月 D 日
> **是否完成周报**：□ 未完成 / □ 已完成

# 已完成工作
Xxxxx

# 下周计划工作
Xxxx
```

**自动生成时预填**：
- 起止日期：从触发指令提取
- 是否完成周报：未完成

**成员填写完成后**：手动改为"已完成"

---

## 4. 命名规则

**周报存档目录结构**：`周报存档 / [年级别节点] / [成员名] / [起止日期]`

| 层级 | 命名 | 示例 |
|---|---|---|
| L1（根） | 周报存档 | （已有节点） |
| **L2（年级别节点）** | `[年份]` | `2026 年`、`2025 年` |
| L3（成员目录） | `[成员名]` | `张三`、`李四` |
| L4（周报文件） | `[起止日期]` | `2026-06-22_2026-06-28` |

**完整示例**：
```
周报存档/
├── 2026 年/                  ← 年级别节点
│   ├── 张三/
│   │   ├── 2026-06-15_2026-06-21
│   │   ├── 2026-06-22_2026-06-28
│   │   └── ...
│   └── 李四/
│       ├── 2026-06-22_2026-06-28
│       └── ...
└── 2025 年/                  ← 年级别节点（去年的归档）
    └── ...
```

**起止日期格式**：`YYYY-MM-DD_YYYY-MM-DD`

**跨年周**：归到**起始日期所在年**（如 2025-12-29_2026-01-04 → `2025 年/` 下）

### 5.1 成员 ↔ 方向

（同 v2.0：方向表为权威维护入口）

### 5.2 周报状态机

```
[自动创建] agent 在「周报收集」下创建子页 [成员名]，内容 = 模板（预填起止日期）
[成员填写] 成员编辑子页，填写"已完成工作"和"下周计划工作"，改状态=已完成
[触发归档] 群里 @ agent"请收 [人名] 的周报（起止日期 ...）"
[归档]     agent 移动子页 → 周报存档 / [年份] / [成员名] / [起止日期]
[下次]     agent 为下个周期的成员创建新的空白周报
```

**关键设计**：归档时**不解析周报文档内容**——起止日期完全由触发指令驱动，避免格式不一致。

## 6. 异常处理

| 异常 | 应对 |
|---|---|
| 触发指令里没起止日期 | 不响应，提示"请提供起止日期（YYYY-MM-DD 到 YYYY-MM-DD）" |
| 触发指令里人名不在成员表 | 跳过该人名，提示"X 不在成员表中" |
| 移动 wiki 节点失败 | 群里报错，让 owner 手动处理 |
| 自动生成空白周报失败 | 群里报错，让 owner 手动创建 |
| 跨年周 | 归到起始年（见 §4） |
| 同名文件已存在 | 在文件名后追加时间戳 `_HHMM`（避免覆盖） |
